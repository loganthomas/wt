Pool mode is here: `wt init` can provision a pool of pre-warmed slots, `wt new`/`wt done` claim and release them with crash-safe leases, `wt claim`/`wt release` expose the plumbing, and `wt pool ls`/`wt pool resize` manage the pool. (#11)
`wt init` now proposes hook and copy defaults by scanning the repo root (lockfiles pick the refresh hook and its gate; untracked `.env` files join the copy list), and pool slots are named `slot-N` for a clean two-level vocabulary: the pool is the collection, a slot is a member. (#11)
